<h1 style="text-align: center;">数据融合在无线传感器网络中的应用</h1>
本文引自<em>袁刚. 无线传感器网络数据融合系统设计[D]. 北京邮电大学, 2009.</em>

由于大多数无线传感器网络应用由大量传感器节点构成，共同完成信息收集、目标监视和感知环境的任务。因此，在信息采集过程中，采用各个节点单独传输数据到汇聚节点的方法并不适合。因为冗余信息会浪费通信宽带，降低信息收集效率，影响信息采集的及时性。

<!--more-->

为避免上述问题，采用数据融合技术，组合出更高效、更符合用户需求的数据。其作用是：
<ul>
 	<li>节省能量</li>
 	<li>获取更准确的信息</li>
 	<li>提高数据收集效率</li>
</ul>

<h3>二、无线传感器网络数据融合需要考虑的问题</h3>
<h4>2.1 考虑资源限制</h4>
考虑到传感器本身的硬件条件：处理能力、内存大小、存储容量、射频能力、电池容量等。
<h4>2.2 考虑数据的有效性</h4>
针对感兴趣的数据进行融合，去掉冗余信息。通常在不同的系统、应用环境中对有用信息的要求不同。
<h4>2.3 考虑数据的实时性</h4>
无线传感器网络通常收集到大量的数据，如果处理不当会造成数据传输延迟，所以要求数据融合系统设计要合理，不能有冗余的处理模块，同时要考虑数据的时间对准问题。
<h4>2.4 人机交互</h4>
人为控制传感器探测目标，从而获取感兴趣的信息。因此这个过程中涉及到，对什么数据感兴趣，对什么样的数据进行融合，调整数据值的有效范围、门限值等。
<h4>2.5 系统模型</h4>
设计合理的系统模型，从而提高系统性能，优化系统资源，增强系统的抗干扰性和应变能力能。

通常数据融合系统分为三个层次，即三个级别的数据融合：像素级、特征级和决策级。

第一层像素级融合需要考虑到对传感器原始数据的处理、数值分析、门限判定、关联性分析等等。同时需要去除数据冗余，降低不必要的无线传输次数和数据以节省能量。

第二层处理中提取测量对象的特征信息，然后把每种测量数据归为一类。经过第二层处理后，数据具有了当前状态参数。

第三次处理针对环境变化趋势，关联性等作出决策，判断有没有危险，并把综合报告作为人机接口的内容显示给观测值，供其决策参考。这个层级的处理需要针对特定的应用场景确定危险评估的方法，确定决策机制。
<h3>三、具体数据融合系统的实现</h3>
<h4>3.1 数据融合算法及其性能分析</h4>
<h5>3.1.1 均值法</h5>
在很多时候，均值算法对获取检测对象的大体状态已经足够了，这时没有必要选择一个复杂度较大的算法进行数据处理。例如，在测量仓储温度时，很多时候仅需指导环境的平均温度即可，即使发生了火灾，再加上一个极值即可。下图展示了数据融合前传感器测温和数据融合后的均值。

<img class="aligncenter size-full wp-image-94" src="http://47.93.1.79/wordpress/wp-content/uploads/2017/09/QQ截图20170905092818.png" alt="" width="558" height="691" />

从两图的对比中可以发现，均值曲线更为平滑，并且使用均值完全可以反映出系统环境的测量参数特征。该融合效率为1/N，这里N为4，即每4个时间节点求其均值。可见在数据平滑度较好的环境中，融合效率很高。但是在动态性比较强的系统中，这个方法就会有较大的误差。
<h5>3.1.2 欧式距离</h5>
欧式距离是为了估计测量误差，同时评估环境中不同区域的差别。当环境的测量数据比较平滑时，应该关闭欧式距离的计算，而当测量值大于阈值时，应该开启欧式距离计算。
<h5>3.1.3 贝叶斯估计</h5>
贝叶斯估计主要应用于系统测量数据连续性好的系统中（因为这是统计学方法，需要长时间序列的数据作为统计基础），其作用是剔除噪音。贝叶斯估计对符合高斯分布的数据进行降噪处理，去除噪音污染，适用于静态估计。
<h5>3.1.4 卡尔曼滤波</h5>
卡尔曼滤波可排除随机干扰，预测下次采集到的数据量，适用于动态变化的系统中，测量各种具有噪声干扰的复杂环境。
<h4>3.2 数据融合树路由</h4>
<ul>
 	<li>兴趣散布阶段：对于传感器网络而言，源节点必须知道目的节点所需的数据类型才能转发相应的数据。因此首要的任务是散布兴趣，让每个节点都知道目标节点的位置信息及兴趣范围。</li>
 	<li>数据融合树构造阶段：该文在DDSP（Destination——driven Shortest path）算法的基础上构造数据融合树。该阶段的主要任务是构造基于组播树的虚拟骨干网。</li>
 	<li>数据传输阶段：探测到节点感兴趣数据的传感器沿着自己的最短路径传输数据。其详细步骤是：当组播树构造完成后，每个传感器节点都将探测到的数据转发给父节点。如果该父节点仅有一个子节点，便直接将数据转发到上级节点，不对数据进行融合；如果该父节点有多个子节点，便等待合适的时间，直到所有子节点都将数据转发给他为止，然后进行数据融合，最后进行转发。</li>
</ul>
